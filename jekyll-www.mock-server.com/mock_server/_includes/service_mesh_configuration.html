Service Mesh / Sidecar Configuration:

These properties configure MockServer for running as a Kubernetes sidecar with transparent proxy interception and xDS route discovery. See Service Mesh / Sidecar Mode for a full usage guide.

Enable transparent HTTP proxy mode where all connections are treated as proxy requests using the Host header as the forwarding target. This enables iptables REDIRECT-based interception without requiring clients to send explicit HTTP CONNECT requests or configure proxy settings.

When enabled, MockServer reads the Host header from each incoming request to determine the forwarding target. If an expectation matches, MockServer returns the mock response; otherwise it forwards to the original target.

Type: boolean Default: false

Java Code:

ConfigurationProperties.transparentProxyEnabled(boolean enable)

System Property:

-Dmockserver.transparentProxyEnabled=...

Environment Variable:

MOCKSERVER_TRANSPARENT_PROXY_ENABLED=...

Property File:

mockserver.transparentProxyEnabled=...

Example:

-Dmockserver.transparentProxyEnabled="true"

Enable the xDS route discovery endpoint at GET /mockserver/xds/routes. When enabled, this endpoint returns active expectations as a simplified xDS RouteConfiguration JSON structure that can be consumed by custom tooling or monitoring dashboards.

Type: boolean Default: false

Java Code:

ConfigurationProperties.xdsEnabled(boolean enable)

System Property:

-Dmockserver.xdsEnabled=...

Environment Variable:

MOCKSERVER_XDS_ENABLED=...

Property File:

mockserver.xdsEnabled=...

Example:

-Dmockserver.xdsEnabled="true"

The port for the xDS route discovery endpoint. Currently reserved for future gRPC-based xDS support. The REST endpoint at /mockserver/xds/routes is served on the main MockServer port.

Type: integer Default: 18000

Java Code:

ConfigurationProperties.xdsPort(int port)

System Property:

-Dmockserver.xdsPort=...

Environment Variable:

MOCKSERVER_XDS_PORT=...

Property File:

mockserver.xdsPort=...

Example:

-Dmockserver.xdsPort="18000"
